# SOLID principle

## Single Resposilbity Principle (SRP)
This principle states that ```a class should have only one reason to change``` which means every class should have a single responsibility or single job or single purpose.

## Open Closed 
This principle states that ```software entities (classes, modules, functions, etc.)``` should be open for extension, but closed for modification‚Äù which means you should be able to extend a class behavior, without modifying it.
